# CLAUDE.md — RealLoveGame

이 파일은 RealLoveGame 레포에서 코드 에이전트(Claude Code, 앤비 등)가 따라야 할 상시 지침이다.
변경이 필요하면 대표자(지노, jino@cv-3.com)의 확인을 먼저 받는다.

---

## 1. 프로젝트 정의

RealLoveGame은 "실생활 연애 시뮬레이션" 웹 MVP다.
사용자가 카카오톡 대화(.txt)를 업로드하고 현재 상황을 적으면, AI가 상황에 맞는 답장 선택지와 각 선택지의 예측 효과를 게임처럼 보여준다.

**이 프로젝트는 제작 테스트용이다.** 재미있고 공감되고 약간 도발적인 톤이 목표이다. 실환경 배포 품질은 목표로 하지 않으며, 드라마틱한 결과도 허용한다.

---

## 2. 핵심 사용자 입력

다음 4가지가 한 세션의 입력이다.

1. **관계 유형 (고정 3종)**: `배우자` / `애인` / `썸`
   - UI에서 라디오 버튼 또는 세그먼트 컨트롤로 선택한다.
   - 이 값에 따라 프롬프트와 해석 톤이 달라진다.
2. **상대방 성향 (E/I × F/T 2축, 총 4종)**: `감성외향(EF)` / `감성내향(IF)` / `이성외향(ET)` / `이성내향(IT)`
   - 드롭다운 또는 4칸 그리드로 선택한다.
   - "잘 모름" 옵션도 허용한다. 이 경우 중립 기준으로 분석한다.
3. **카카오톡 대화 파일(.txt)**: 클라이언트에서만 처리한다. 서버 업로드·저장 금지.
4. **현재 상황 텍스트**: 예) "오빠가 '알았어 말어' 하고 답장이 끊겼어."

## 3. 기술 스택

- **프레임워크**: Next.js (App Router)
- **스타일**: TailwindCSS
- **언어**: TypeScript
- **AI 모델**: Google Gemini API (`gemini-2.5-flash` 기본, 필요 시 `gemini-2.5-pro`)
  - **⚠️ 이 프로젝트는 제작 테스트용이다. 실환경 배포 품질을 목표로 하지 않는다.**
  - SDK: `@google/generative-ai`
  - 환경변수: `GEMINI_API_KEY` (`.env.local`에만 둔다. 클라이언트 노출 절대 금지)
- **배포**: Vercel
- **DB**: 없음 (MVP). 모든 데이터는 세션 내 휘발.
- **언어(UI/프롬프트)**: 한국어 단일. i18n은 도입하지 않는다.

## 4. 카카오톡 파싱 규칙

카톡 export 파일은 다음 형식을 따른다고 가정한다.

```
[이름] [오전 10:20] 메시지
[이름] [오후 3:05] 메시지
날짜 구분선: "--------------- 2026년 4월 10일 토요일 ---------------"
시스템 메시지: "○○님이 나갔습니다" 등
```

파서(`/lib/parser/kakao.ts`)는 다음을 추출한다.

- 최근 메시지 200줄 (너무 길면 토큰 낭비)
- 사용자 이름 추정 (내보낸 본인 이름은 파일 헤더에 있음)
- 상대방 이름 추정
- 시스템 메시지·날짜 구분선은 제거
- ~~tone_summary 휴리스틱~~ → **제거. recent_snippet에 집중한다.**

**파일 자체는 서버로 보내지 않는다.** 서버로 가는 것은 `{ recent_snippet, relation_type, partner_tendency, user_situation }` 뿐이다.

## 5. API 라우트

### `POST /api/simulate-love`

**요청 바디**
```ts
{
  relation_type: "배우자" | "애인" | "썸",
  partner_tendency: "EF" | "IF" | "ET" | "IT" | "UNKNOWN",
  recent_snippet: string, // 최근 대화 요약본 (클라이언트 파싱 결과)
  user_situation: string
}
```

**응답 바디**
```ts
{
  options: Array<{
    emoji: string,
    reply_text: string,
    affection_change: number,        // -5 ~ +5
    anger_level: number,             // 0 ~ 10
    continuation_probability: number, // 0 ~ 100
    risk_level: "낮음" | "중간" | "높음",
    short_prediction: string          // 1~2문장 한국어
  }>,
  best_index: number,   // options 중 추천
  worst_index: number   // options 중 비추천
}
```

**선택지는 항상 정확히 3개다.** AI에게 3개를 강제하고, 클라이언트도 3개 고정 레이아웃으로 렌더한다.

---

## 6. AI 프롬프트 원칙

Gemini에게 다음을 지킨다.

- 시스템 프롬프트에 **관계 유형과 성향(EF/IF/ET/IT)을 명시**해 해석 기준을 고정한다.
- 응답은 반드시 JSON Schema에 맞춘 구조화 출력(`responseMimeType: "application/json"`)으로 받는다.
- 톤: 친근한 한국어, 약간 과장된 게임 내레이션 톤. 반말 허용. 단 욕설 금지.
- 선택지는 **항상 3개**, 명확한 대비를 만든다 (예: 다정형 / 쿨한척형 / 직진형). 밋밋한 3개는 금지.
- `short_prediction`은 "그러면 어떻게 될지"를 구체적이고 드라마틱하게 묘사한다. 뻔한 말 금지.
- **이 프로젝트는 웃긴 테스트용이다.** 약간 과장되고 드라마틱해도 된다. 관계가 망가지는 선택지도 있어도 된다.

## 7. 가드레일 (필수)

AI 출력과 UI는 다음을 절대 생성·허용하지 않는다.

- 감정 학대, 가스라이팅, 조종(manipulation) 기법을 권장하는 답변
- 관계 단절을 충동적으로 유도하는 답변 (이별·이혼을 "게임 점수" 식으로 가볍게 처리 금지)
- 자해·자살·폭력 암시
- 상대방 외모·가족·직업 비하
- 허위·과장 (예: "이 답장 하면 100% 돌아옵니다")
- 경쟁 서비스 비방
- 성적으로 노골적인 표현

민감 상황(가정폭력, 자해 징후 등)이 감지되면 AI는 시뮬레이션 대신 전문 상담 안내를 내보낸다.

---

## 8. 개인정보 보호

- 카톡 파일 원문은 **서버에 절대 올리지 않는다.** 서버 로그에도 남기지 않는다.
- 서버로 가는 요약 데이터도 저장하지 않는다. API 라우트는 stateless하게 짠다.
- 화면 하단에 항상 다음 문구를 고정 노출한다.
  > "업로드한 대화는 저장되지 않습니다. 브라우저에서만 처리되며, 시뮬레이션에는 요약된 일부 정보만 사용됩니다."
- 이름·전화번호 등 개인정보가 요약에 섞여 들어갈 가능성이 있으므로, 파서 단계에서 이름을 `A`, `B`로 치환한 뒤 서버로 보낸다.

---

## 9. UI/UX 원칙

- **5초 안에 뭔지 이해되게** 만든다.
- 메인 플로우: `파일 업로드 → 관계·MBTI 선택 → 상황 입력 → [시뮬레이트] 버튼 → 선택지 카드들`
- 선택지는 카드 UI로 나열한다. 각 카드에 이모지, 예상 스탯(애정/분노/지속확률/리스크), 한 줄 예측을 담는다.
- **Best / Worst 카드는 배지로 강조한다.**
- 모바일 반응형 필수 (절반 이상이 모바일에서 쓸 것을 가정).
- 로딩 상태는 드라마틱하게 처리한다. 예시 문구 풀:
  - "읽씹 확률 계산 중…"
  - "지금 답장하면 생존 가능성 분석 중…"
  - "상대방 마음 스캔 중…"
  - "답장 3초 뒤 미래 시뮬레이션 중…"
  - "이 선택의 나비효과 추적 중…"
- 에러는 유저 친화적 한국어 메시지로. 스택 트레이스 노출 금지.

## 11. 모델 사용 기준 (상위 워크스페이스 지침 연동)

- **앱 내부에서 호출하는 모델은 Gemini Flash 고정** (토큰·비용 절약).

## 12. 개발 규칙

- **타입 안전**: API 요청/응답은 zod로 런타임 검증 후 타입 추론한다.
- **환경변수**: `GEMINI_API_KEY`는 `process.env.GEMINI_API_KEY`로만 접근. `NEXT_PUBLIC_` 접두사 절대 금지.
- **의존성 최소화**: MVP이므로 상태 관리 라이브러리, 인증, DB 드라이버 등 불필요한 패키지는 넣지 않는다.
- **린트·포맷**: `eslint`, `prettier` 기본 세팅만 한다.
- **테스트**: 해커톤 범위에서는 파서 단위 테스트 1~2개만. E2E는 생략.
- **커밋 단위**: 기능별로 쪼갠다.

## 13. 빌드·배포

- 로컬: `npm run dev`
- 빌드: `npm run build`
- 배포: Vercel. `GEMINI_API_KEY`는 Vercel 환경변수에 등록한다.
- 도메인: 해커톤 범위에서는 Vercel 기본 도메인을 사용한다.

## 14. 금지사항 요약 (코드 에이전트용)

- 카톡 원문을 서버·DB·로그 어디에도 쓰지 않는다.
- Gemini 외 다른 LLM API를 함부로 붙이지 않는다. 필요 시 대표자 확인.
- 관계 파괴·감정 학대를 유도하는 프롬프트·응답을 만들지 않는다.
- API 키를 클라이언트 번들에 노출하지 않는다.
- 과장·허위 문구("100% 돌아옵니다" 등) 생성을 허용하지 않는다.
